Configure Ubiquiti Dream Machine for Multiple VLANs and Wi-Fi Networks

This guide provides detailed instructions on how to configure the Ubiquiti Dream Machine (UDM) to support multiple VLANs and associated Wi-Fi SSIDs to segment network traffic for management, user devices, IoT devices, guest access, and streaming services.

Prerequisites

  • Ubiquiti Dream Machine (UDM)
  • Ubiquiti Access Points (APs) compatible with UDM
  • Network cables
  • Access to UniFi Network application (latest version)

1. VLAN Configuration

Begin by configuring the VLANs on your UDM. VLANs help in segmenting the network which enhances security and improves network management.

Step 1: Access the UniFi Network Application

  1. Connect your device to the network managed by the UDM.
  2. Open a web browser and navigate to https://unifi.ui.com or the IP address assigned to your UDM.
  3. Log in using your Ubiquiti account credentials.

Step 2: Create VLANs

Navigate to the settings area and select 'Networks'. Click on 'Create New Network' and configure each VLAN as follows:

VLAN Name VLAN ID Purpose
WYSECHOICE_MGMT 10 Management
WYSECHOICE 40 User devices
WYSECHOICE_IOT 20 IoT Devices
WYSECHOICE_GUEST 50 Guest Access
WYSECHOICE_STREAMING 30 Streaming Devices

5. Advanced Network Settings

Advanced settings allow you to further tailor network performance and security according to specific needs. Here are some of the critical advanced configurations:

Step 1: Enable IGMP Snooping

For networks handling multicast traffic, such as WYSECHOICE_STREAMING:

  1. Navigate to 'Networks'.
  2. Select the WYSECHOICE_STREAMING VLAN.
  3. In the 'Edit Network' pane, find 'Advanced' and enable IGMP Snooping to improve network efficiency with multicast traffic.

Step 2: Apply Firewall Rules

To enhance network security, define firewall rules that control traffic flow between VLANs:

  1. Go to 'Settings' > 'Internet Security' > 'Firewall'.
  2. Choose 'LAN IN' to create rules for incoming traffic to your LAN networks.
  3. Create rules such as blocking all traffic from WYSECHOICE_GUEST to WYSECHOICE_MGMT to prevent guest access to management interfaces.
  4. Ensure to specify source and destination VLANs, action to take (accept, drop), and apply the changes.

Step 3: Quality of Service (QoS)

If bandwidth-intensive applications are used, configure QoS to prioritize traffic:

  1. Navigate to 'Settings' > 'Networks' and select the network (e.g., WYSECHOICE).
  2. In the 'Edit Network' section, scroll to 'Advanced Options' and enable QoS.
  3. Set bandwidth limits or prioritize by application or device type.

6. Security Enhancements

Enhancing security is paramount, especially with multiple network segments and external guest access.

Step 1: Enable Threat Management

UDM's threat management integrates with UniFi's security gateway services to provide intrusion detection and prevention:

  1. Go to 'Settings' > 'Threat Management'.
  2. Enable 'Intrusion Detection System' and 'Intrusion Prevention System' for real-time threat detection and action.
  3. Adjust the sensitivity level based on your network environment and traffic.

Step 2: Use Secure Protocols

Ensure all management interfaces and device communications are secured through protocols like HTTPS, SSH, and SNMPv3:

  1. Access 'Settings' > 'Services'.
  2. Enable HTTPS for web interface, SSH for secure command-line access, and configure SNMPv3 for network management monitoring.

7. Troubleshooting Common Issues

Here are troubleshooting steps for common issues that might arise after configuration:

Connectivity Issues

If devices are unable to connect or obtain an IP address:

  1. Check if the DHCP server is enabled and properly configured on each VLAN.
  2. Ensure wireless networks are associated with the correct VLAN.
  3. Verify physical connections and AP statuses through the UniFi Network application.

Performance Issues

For slow network speeds or poor connectivity:

  1. Examine the network utilization and interference using the UniFi Network application tools.
  2. Adjust the QoS settings to prioritize critical traffic.
  3. Check for firmware updates for UDM and APs, as updates can include performance enhancements.

Security Concerns

If there are unexpected access or security alerts:

  1. Review firewall rules and ensure they are correctly applied to each VLAN.
  2. Check the logs in 'Threat Management' for any detected threats and address them accordingly.
  3. Update passwords and security settings regularly to strengthen network security.

8. Regular Maintenance and Documentation

Maintaining network health and documentation is crucial for long-term stability and troubleshooting:

    Periodically review and update the network configuration to adapt to new requirements or changes in the network environment.
  1. Regularly back up the UniFi Controller settings to ensure quick recovery in case of hardware failure or corruption.
  2. Document changes and keep a version-controlled log of configurations to track historical adjustments and their impacts.
